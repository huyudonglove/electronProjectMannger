# Electron Manager UI 设计系统

## 产品定位

Electron Manager 是项目上下文与 Agent 协作工作台，不是传统后台，也不是单纯的数据看板。界面优先帮助用户判断：当前项目发生了什么、下一步要处理什么、Agent 是否需要接续。

视觉方向：Calm Workbench（安静、密度适中、信息有层级）。

## 视觉规则

- 深色工作台是默认主题，浅色主题复用同一组语义 Token。
- 暗色模式使用中性黑灰，亮色模式使用暖白；两种主题都以 `#78461A` 作为唯一品牌主色，不使用蓝色背景、蓝色选中面或蓝色主按钮。
- 编号、状态和分类默认使用普通辅助文字；只有告警、阻塞等异常状态使用语义色圆点。
- 琥珀只表示需要关注，淡灰表示完成或已处理，红色只表示错误或危险；完成态不使用绿色。
- 使用 8px 间距基线；页面间距 24px，卡片内距 16px，紧凑区域内距 12px。
- 卡片圆角 10px，控件圆角 6px，弹窗圆角 14px。
- 页面最多使用三层信息：主内容、操作元信息、关联元信息。
- 重要状态使用徽标；次要信息使用小圆点加文字；关联 ID、来源和版本使用弱化文本。
- 不使用彩色标签堆叠表达所有字段，不让装饰性信息抢过标题和待办内容。

## 页面结构

侧边栏分为三组：

1. 工作区：总览、想法、任务、研究、协作。
2. 资料库：文档、知识库。
3. 管理：版本、工作记录、约束。

顶部只放项目、版本、刷新、项目切换和主题操作。页面内容统一使用：页面标题、上下文说明、主要内容区、空状态。

## 页面职责

- 总览：先显示当前状态、待处理事项和活跃任务，统计数字作为辅助信息。
- 任务：使用看板；任务卡突出标题、状态、优先级线和验收摘要。
- 研究：使用队列；待研究事项优先，已完成研究提供结果和关联文档。
- 协作：使用收件箱；明确区分“待我回复”和“待 Agent 跟进”。
- 工作记录：左侧目录、右侧详情；日志级别是元信息，不应压过日志内容。
- 文档与知识库：统一使用资料卡片和详情视图，仅在来源和操作上区分。
- 约束：项目规则和系统规则分层展示，系统规则只读。

## 组件边界

基础组件只负责视觉和交互状态，不读取项目数据：

- `UiButton`、`UiIconButton`
- `UiBadge`、`UiMeta`
- `UiCard`、`UiListRow`
- `UiTabs`、`UiDialog`
- `UiInput`、`UiSelect`、`UiTextarea`
- `UiEmptyState`、`UiSectionHeader`

业务视图负责数据组合：

- `OverviewView`
- `TaskBoardView`
- `ResearchView`
- `CollaborationView`
- `LogExplorerView`
- `LibraryView`

基础组件不得依赖 `Dashboard`、项目路径或具体记录类型；业务视图不得重复定义按钮、徽标、卡片和空状态样式。

## 改造原则

- 保持现有 IPC、数据结构和业务行为不变，先改表现层。
- 先 Token 和基础组件，再拆页面，最后删除旧样式。
- 每完成一个页面都运行构建，并检查深色、浅色、窄窗口和空数据状态。
- 新增 UI 优先复用基础组件，不在页面中新增同义 CSS 类。
