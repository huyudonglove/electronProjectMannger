# 旧项目迁移说明

Electron Manager 不在运行时自动迁移或修复旧项目数据。打开项目、查看 Dashboard、刷新引导都不会改写已有 Markdown。

## 新项目

直接通过 Electron Manager 初始化。初始化会创建当前版本的完整目录、模板和派生文件，不需要执行迁移步骤。

## 旧项目

只有需要继续使用旧数据时才迁移，建议按下面步骤手动处理：

1. 先备份旧项目管理数据目录。
2. 在 Electron Manager 中初始化一个新项目，得到当前结构和模板。
3. 按内容复制到新项目：
   - 任务、想法、研究、问题、风险、工作记录 → `versions/V001/` 对应文件。
   - 项目文档 → `documents/`。
   - 全局知识 → 管理数据根目录的 `knowledge/`。
4. 按当前模板补齐必要字段，例如 `version:: V001`、研究的 `mode:: breadth | depth`、问题的 `origin::` 和 `### 对话记录`、工作记录的 `record_level:: light | standard | deep`。
5. 重新打开或刷新项目，确认 Dashboard、Agent brief 和当前版本路径正常。

不要直接复制旧的 `agent-brief.json`、`index.json`、基线或 `record-counters.json`；这些文件由当前项目初始化和后续写入重新生成。旧格式只需保留原文时，也可以先作为文档放入 `documents/`，再逐条整理为当前记录。

如果旧数据不再需要，直接初始化新项目即可，无需迁移。
