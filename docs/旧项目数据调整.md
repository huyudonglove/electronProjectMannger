# 旧项目数据调整方式

这份文档只用于早期 Electron Manager 项目数据的手动整理。新建项目不需要阅读或执行迁移步骤，直接按 README 中的当前使用方式即可。

Electron Manager 不在代码、解析器、文件查找或界面层长期兼容旧数据结构。旧项目如果是早期目录或早期字段，处理方式是一次性整理 Markdown 文件，让数据变成当前结构；整理完成后，应用只读取当前结构。

## 调整原则

- 不增加运行时代码分支。
- 不增加旧字段别名。
- 不增加旧目录扫描。
- 不做文件层自动搬迁。
- 不做 UI 兜底显示。
- 旧数据只通过这份文档指导手动整理；整理后就按当前数据结构使用。

## 目录映射

| 旧位置 | 当前位置 |
| --- | --- |
| `00_项目管理/工程任务.md` | `tasks/工程任务.md` |
| `00_项目管理/数据层规范.md` | `collaboration/数据层规范.md` |
| `00_项目管理/Agent 同步交接.md` | `collaboration/Agent 同步交接.md` |
| `04_记录库/想法与问题.md` | `thoughts/想法与问题.md` |
| `04_记录库/研究.md` | `research/研究.md` |
| `04_记录库/Agent 工作记录.md` | `work-logs/Agent 工作记录.md` |
| `04_记录库/需求变更索引.md` | `collaboration/需求变更索引.md` |
| `01_知识结构/` 里的知识条目 | 全局 `<app-data-root>/knowledge/` |
| 混在知识库或其他目录里的项目文档 | 项目本地 `<managed-data-root>/documents/` |

## 字段整理

- 任务使用 `short_id:: Txxx`，并按 `Txxx` 倒序摆放，ID 越大越靠前。
- 想法使用 `short_id:: Ixxx`，并按 `Ixxx` 倒序摆放，ID 越大越靠前。
- 研究使用 `short_id:: Dxxx`，保留 `### 内容`、`### 回答`、`### 验收标准`。
- 文档使用 `short_id:: Wxxx`，只表示项目本地文档编号。
- 工作记录使用 `log_short_id:: Lxxx`；关联任务时写 `task_short_id:: Txxx`。
- 知识条目使用 `short_id:: Kxxx`，放到所有项目共享的全局 `knowledge/` 目录，不放在单个项目数据目录下。
- 文档只放在项目本地 `documents/` 目录，文档页不汇总任务、想法、研究、协作或工作记录；文档不会自动进入知识库。

## 工作记录当前格式

工作记录应保留这些段落：

```markdown
### 用户目标
### 需求理解
### 产出
### 关键步骤
### 执行动作
### 验证
### 验收标准
### 未确认事项
```

示例字段：

```markdown
type:: agent-log
log_short_id:: L001
created:: YYYY-MM-DD HH:mm
task_short_id:: T001
```

## 研究当前格式

研究记录应保留这些段落：

```markdown
### 内容
### 回答
### 验收标准
```

示例字段：

```markdown
id:: dialogue-...
short_id:: D001
type:: dialogue
created:: YYYY-MM-DD HH:mm
mode:: learning | research | decision
tags:: learning, research
related_tasks:: T001
related_thoughts:: I001
```

## 知识库调整

知识库是所有项目共享的外层目录，不属于单个项目。

旧项目中 `01_知识结构/` 下真正属于长期知识的 Markdown，应移动到：

```text
<app-data-root>/knowledge/
```

移动后补齐当前知识字段：

```markdown
id:: knowledge-...
short_id:: K001
type:: knowledge
status:: active
created:: YYYY-MM-DD HH:mm
updated:: YYYY-MM-DD HH:mm
tags:: 无
aliases:: 无
source:: 无
related_records:: 无
related_tasks:: 无
related_notes:: 无
summary:: 简短摘要
```

如果文件只是项目文档、说明、手册或普通资料，不要放进知识库，应移动到项目本地：

```text
<managed-data-root>/documents/
```

## 完成检查

- 当前项目数据目录下存在 `tasks/`、`thoughts/`、`research/`、`work-logs/`、`collaboration/`、`documents/`。
- 全局应用数据目录下存在 `knowledge/`。
- 不再依赖 `00_项目管理/`、`01_知识结构/`、`04_记录库/`。
- 记录 ID 按 `T/I/D/W/K/L` 倒序摆放。
- 工作记录有 `log_short_id:: Lxxx`。
- 关联任务的工作记录有 `task_short_id:: Txxx`。
