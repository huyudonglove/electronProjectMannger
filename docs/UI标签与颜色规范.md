# UI 标签与颜色规范

本规范用于约束 Electron Manager 的状态、元信息、短编号和颜色使用。页面新增信息时优先复用这里的语义，不再按页面单独创造 Tag 样式。

## 现状清单

| 页面 | 当前信息 | 当前表现 | 问题 |
| --- | --- | --- | --- |
| 总览 | Ready、Agent 同步 | 普通 Badge | 不是状态，不需要 Tag |
| 任务 | 短编号、优先级、领域、状态、列计数 | 编号块、圆点文字、状态文字、色轨 | 同一张卡片最多出现四种标记；列已经表达状态 |
| 想法 | 短编号、状态 | 编号文字、状态 Badge | 状态缺少统一图标 |
| 研究 | 短编号、模式、状态、关联记录 | 多个 Badge | 模式和状态视觉权重相同 |
| 版本 | 版本号、版本标签、当前状态 | 编号、Badge、状态文字 | 标签与版本标题重复抢占注意力 |
| 协作 | 短编号、问题类型、阻塞、关系、处理状态 | Badge、圆点 Badge、普通文字 | 状态与分类没有明确层级 |
| 工作记录 | 日志号、Done、记录等级、任务号、来源 | 一行多个 Badge/编号 | 目录项顶部信息过密，Done 重复出现 |
| 文档/知识库 | 出处、分类、关联记录 | 弹窗 Badge | 普通元信息被做成状态标签 |
| 约束 | 约束类型、来源 | 图标卡片、文字 | 已较克制，保持元信息样式 |

## 目标色板

基础层使用中性黑灰和暖白，`#78461A` 是全局唯一品牌主色。语义色只用于状态，不使用蓝色背景、蓝色选中面或蓝色主按钮。

| 角色 | 亮色 | 暗色 |
| --- | --- | --- |
| 页面背景 | `#f7f5f2` | `#09090b` |
| 卡片 | `#ffffff` | `#111113` |
| 次级表面 | `#f1ede8` | `#18181b` |
| 正文 | `#211a15` | `#fafafa` |
| 辅助文字/图标 | `#76685d` | `#a1a1aa` |
| 边框 | `#ded6ce` | `#27272a` |
| 品牌主色/主操作 | `#78461A` | `#78461A` |
| 暗色小字/细图标 | — | `#c78a52` |
| 完成/已处理 | `#71717a` | `#a1a1aa` |

## Tag 规则

### 1. 短编号

`T113`、`L263`、`R041`、`V002` 只使用等宽辅助文字，不使用背景、边框或圆角。

### 2. 元信息

领域、来源、记录等级、研究模式、关联关系使用无底色的“图标 + 文字”。

- 分类/领域：Tag 图标。
- 记录等级：轻量使用圆点，标准使用层级，深度使用搜索。
- 研究模式：广度使用分支，深度使用搜索。
- 关联关系：链接图标。

记录详情的左侧层级轨使用同一品牌色递进：轻量为 2px 弱色、标准为 3px 中色、深度为 4px 主色；记录等级不借用成功、警告或危险色。

### 3. 状态

状态使用“状态图标 + 文字 + 语义色”。只有状态可以使用弱背景和小圆角，禁止胶囊形。

| 状态 | 图标 | 颜色 |
| --- | --- | --- |
| Todo / 待处理 | 空心圆点 | 中性 |
| Doing / 待 Agent 跟进 | 时钟 | 警告 |
| Done / 已完成 / 已处理 | 圆形勾选 | 淡灰 |
| 阻塞 / 风险 | 警告三角 | 警告 |
| Abandoned / 失败 | 圆形叉号 | 危险 |

### 4. 冗余信息

- 任务卡片不重复显示所在列已经表达的状态。
- 工作记录目录默认不重复显示全部为 Done 的状态；只在非完成状态时展示。
- 卡片左侧状态轨与状态图标不能同时使用多种颜色表达同一件事。
- 单行最多展示两个元信息标记；其余放入详情。
